用語
リポジトリ：コードやファイル、ディレクトリの内容と変更履歴を記録しておく場所。
リモートリポジトリ：githubなどのインターネット上にあるリポジトリ。
ローカルリポジトリ：パソコンのストレージなどのローカルストレージにあるリポジトリ
ブランチ：履歴を分岐して記録する履歴の枝葉。１つのプロジェクトからブランチさせる事でお互いの履歴に干渉しない。
ローカルブランチ：ローカルリポジトリで記録されるブランチ
リモートブランチ；リモートリポジトリで記録されるブランチ
デフォルトブランチ：コミットを行う基点となるブランチ
ワークツリー：実際に作業しているプロジェクトやファイルなど
ステージ：ワークツリーで変更したファイルなどを追加する場所

git command
clone：リモートリポジトリから複製する
commit：ステージのファイルをリポジトリに送信
branch：ブランチを一覧表示
branch <ブランチ名>：新規作成する
status：ステージに追加される前の現在の変更状況を確認
add：ワークツリーで変更したファイルをステージに追加
diff：ファイルの変更された内容を確認
diff --staged：リポジトリと比較したステージで変更されたファイルの内容を確認
log：変更履歴の確認
push：ローカルリポジトリの内容をリモートリポジトリに送信
checkout --：ステージにある情報を元にワークツリーのファイルの変更を取り消す
checkout <ブランチ名>：ブランチの切り替え
fetch：リモートリポジトリからローカルリポジトリに情報を反映
pull：リモートリポジトリからローカルリポジトリ―とワークツリーに反映
marge:変更内容を一つのコミットに取り込む




バージョン管理
.gitignore：.gitignoreファイルにgitに管理されたくないファイルやディレクトリを指定して、管理から除外
